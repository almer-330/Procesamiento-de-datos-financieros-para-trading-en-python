02_datos_tiempo_real_wallstreet.py — Procesamiento de Datos Financieros
¿Qué vamos a lograr en esta lección?
contexto Muchos sistemas de trading necesitan saber ahora mismo cuánto vale un activo. No el precio de ayer, no el de hace una hora — el precio en este momento. Esta lección te enseña a pedirlo con una sola línea de Python.
Las tres herramientas que usaremos
librería principal
wallstreet
Obtiene datos de mercado en tiempo real. Su nombre viene de Wall Street, el famoso centro financiero de Nueva York.
librería de concurrencia
threading
Permite ejecutar tareas "en paralelo" — el programa puede hacer otras cosas mientras espera el siguiente precio.
librería de tiempo
time
Pone a "dormir" el programa N segundos entre cada consulta, para no bombardear el servidor con peticiones continuas.
instalación
pip install wallstreet
Ejecuta en tu terminal de Anaconda o en cualquier entorno donde tengas Python activo.
Los atributos clave del objeto Stock
stock.price
precio actual
Precio de mercado en este momento, en la moneda nativa del activo.
stock.name
nombre del activo
Nombre completo de la empresa (ej. "Apple Inc.").
stock.currency
moneda
Moneda en la que cotiza: USD, EUR, MXN, etc.
stock.change
cambio en $
Cuánto subió o bajó el precio desde que abrió el mercado hoy.
stock.cp
cambio en %
"Change Percent" — el mismo movimiento expresado como porcentaje.
stock.quote
ticker / símbolo
El símbolo bursátil que usaste al crear el objeto (ej. "AAPL").
¿qué es un ticker? Es el código abreviado con el que se identifica una empresa en bolsa. Apple → AAPL, Amazon → AMZN, Tesla → TSLA, Microsoft → MSFT. Es como el "usuario" de la empresa en el mercado.
Código paso a paso — parte 1: petición simple
paso 1 — importar librerías
import wallstreet # datos de mercado en tiempo real
import threading # para ejecutar en paralelo
import time # para pausar entre peticiones
¿por qué estas tres? wallstreet hace el trabajo pesado (pedir el precio). threading deja que esa petición corra en segundo plano mientras el resto del programa sigue vivo. time controla la cadencia — sin sleep(), consultaríamos miles de veces por segundo.
paso 2 — crear el objeto Stock
ticker = "AAPL" # símbolo de Apple
stock = wallstreet.Stock(quote=ticker) # petición al servidor
¿qué pasa aquí? Al escribir wallstreet.Stock(quote="AAPL") Python sale a internet en ese momento, consulta el precio actualizado de Apple y guarda toda la información en el objeto stock. Es una sola línea que reemplaza decenas de líneas de código con requests HTTP.
paso 3 — mostrar precio, cambios y moneda
print(f"Precio de {stock.name}: {stock.price} en {stock.currency}")
print(f"Cambio %: {stock.cp:.2f}%")
print(f"Cambio $: {stock.change:.2f}")
el :.2f Dentro de las llaves {} de un f-string, :.2f significa "muestra este número con 2 decimales". Sin esto, Python podría imprimir 0.3700000000001 en lugar de 0.37.
paso 4 — calcular el precio de cierre de ayer
precio_anterior = stock.price - stock.change
print(f"Cierre de ayer: {precio_anterior:.2f}")
lógica del cálculo
precio_ayer = precio_hoy − cambio_de_hoy
Si AAPL está en $238.50 y subió $1.20 hoy → ayer cerró en $237.30
# salida esperada en consola
Precio de Apple Inc.: 238.50 en USD
Cambio %: 0.51%
Cambio $: 1.20
Cierre de ayer: 237.30
Parte 2 — monitoreo automático en paralelo
¿por qué necesitamos hilos? Si consultamos el precio en un bucle normal (while True), el programa queda bloqueado ahí — no puede hacer nada más mientras espera. Con threading, ese bucle corre en un "carril separado" y el programa principal sigue libre.
La variable de control
detenerse = False # interruptor: False = seguir, True = parar
interruptor on/off Esta variable actúa como un interruptor de luz. Mientras sea False, el hilo sigue corriendo. Cuando la cambiamos a True desde cualquier parte del código, el hilo termina limpiamente en la próxima iteración.
La función de monitoreo
def Obtener_Precio(tickers: list, n_segundos: float):
    """Consulta el precio de cada ticker cada n_segundos."""
    while True:
        for ticker in tickers:
            t = wallstreet.Stock(quote=ticker)
            print(f"{t.name}: {t.price} {t.currency} | Cambio: {t.change:.2f}")
        if detenerse:
            break
        time.sleep(n_segundos)
Lanzar y controlar el hilo
tickers = ["AMZN", "TSLA", "MSFT"]

hilo = threading.Thread(
    target = Obtener_Precio, # función a ejecutar
    args = (tickers, 10) # argumentos: lista + 10 segundos
)
hilo.start() # dispara el hilo

# para detenerlo:
detenerse = True
¿Qué sucede en el tiempo?
línea de tiempo de ejecución
programa principal
define variables
inicia hilo
detenerse = True
hilo de fondo
consulta AMZN TSLA MSFT
sleep(10s)
consulta otra vez
if detenerse → break
Ambos "carriles" corren al mismo tiempo sin bloquearse mutuamente.
error común en la lección El instructor olvidó pasar el segundo argumento (n_segundos) la primera vez que ejecutó. El error que Python da es: TypeError: Obtener_Precio() missing 1 required positional argument. La solución fue agregar el 10 en el args=(tickers, 10).
nota sobre el ejemplo del curso En el código ejecutaron detenerse = True inmediatamente después de hilo.start(), por lo que el hilo solo hizo una iteración antes de detenerse. En una aplicación real, esa línea se pondría en un botón o condición separada para dejar que corra continuamente.

Lo que tienes que dominar de esta lección
▸
wallstreet.Stock(quote="TICKER") — una sola línea que conecta con el mercado y devuelve un objeto con precio, cambio, moneda y nombre del activo.
▸
Atributos del objeto Stock: .price (precio actual), .change (cambio en $), .cp (cambio en %), .currency, .name.
▸
F-strings con formato: {variable:.2f} redondea a 2 decimales dentro del print. Es la forma estándar de formatear precios.
▸
Cálculo del precio anterior: precio_ayer = stock.price − stock.change. El precio actual ya incluye el movimiento de hoy.
▸
threading.Thread(target=función, args=(...)) — lanza una función en un hilo separado para que corra en paralelo. Se activa con .start().
▸
Variable de control: un booleano global (detenerse) actúa como interruptor para terminar el hilo limpiamente desde fuera de la función.
▸
time.sleep(n) — pausa la ejecución N segundos. Sin esto, el bucle haría miles de peticiones por segundo innecesariamente.
