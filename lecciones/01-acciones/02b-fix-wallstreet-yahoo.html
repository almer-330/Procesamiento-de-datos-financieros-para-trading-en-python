02b_fix_wallstreet_yahoo.py — Procesamiento de Datos Financieros
¿Qué está pasando y por qué falla el código?
situación Después de instalar wallstreet y escribir el código de la lección anterior, al ejecutarlo aparece un error. El código no cambió — lo que cambió fue algo externo: la API de Yahoo Finance.
El código que falla
import wallstreet
ticker = "AAPL"
stock = wallstreet.Stock(quote=ticker)
print(stock.price)
El error que aparece en consola
YFRateLimitError: Too many requests.
You must either use a session that handles its own cookies
or let the Yahoo Finance API manage its own sessions.
# → "Deja que Yahoo Finance gestione sus propias sesiones"
¿qué significa ese mensaje? Yahoo Finance está diciendo: "no me mandes una sesión de requests — yo me encargo de eso". La librería wallstreet estaba creando su propia sesión HTTP y pasándosela a Yahoo Finance, pero la API dejó de aceptar eso. La solución es simplemente no pasar ninguna sesión — dejar que Yahoo Finance use la suya propia.

¿Por qué ocurre esto? — dependencias y actualizaciones
dependencias en cadena wallstreet no habla con Yahoo Finance directamente. Por dentro usa yfinance (otra librería). Cuando Yahoo actualizó su API, yfinance tuvo que adaptarse — pero wallstreet aún no actualizó su código interno para seguir esa adaptación.
Las tres capas involucradas
wallstreet
Librería que usamos. Todavía pasa session=requests.Session() internamente — esto ya no funciona.
yfinance
Librería intermedia. Se actualizó para pedir que la sesión sea None — pero wallstreet aún no le hace caso.
Yahoo Finance API
El servidor externo. Actualizó su protocolo y ya no acepta sesiones externas — quiere gestionar sus propias cookies.
lección de programación Esto pasa todo el tiempo en el mundo real. Cuando una librería depende de otra que depende de un servicio externo, cualquier cambio en la cadena puede romper tu código aunque tú no hayas cambiado nada. Por eso existe el control de versiones y los archivos requirements.txt.
¿cómo localizar el archivo de wallstreet? En la lección, el instructor ejecutó una línea para encontrar dónde Python instaló el módulo:

import wallstreet; print(wallstreet.__file__)

Esto imprime la ruta completa del archivo .py de la librería — por ejemplo:
C:\Users\alan\anaconda3\envs\trading\Lib\site-packages\wallstreet\wallstreet.py
Paso a paso para corregir el error
importante antes de empezar Vas a editar directamente el archivo de una librería instalada. No es algo que se hace frecuentemente — es un parche temporal hasta que los autores de wallstreet publiquen una versión actualizada. Guarda una copia del archivo original si quieres ser precavido.
1
Localizar el archivo de wallstreet
Ejecuta esto en Spyder o en tu terminal para obtener la ruta del archivo:
import wallstreet
print(wallstreet.__file__)
Copia la ruta que aparece en consola (sin el nombre del archivo final) y pégala en el explorador de archivos de tu sistema.
2
Abrir el archivo wallstreet.py
Navega hasta la carpeta que apareció y abre wallstreet.py con cualquier editor de texto (Notepad++, VS Code, o el propio Spyder).
3
Localizar la línea problemática en la clase Stock
Busca dentro del archivo la clase Stock y dentro de ella la línea donde se pasa session=. Verás algo como esto:
# busca una línea similar a esta:
data = yf.download(..., session=requests.Session())
4
Hacer el cambio — reemplazar la sesión por None
Este es el único cambio que necesitas hacer:
wallstreet.py — clase Stock
− session=requests.Session()
+ session=None  # deja que yfinance gestione su propia sesión
Solo esa palabra: None en lugar de requests.Session(). Guarda el archivo.
5
Reiniciar la terminal / kernel de Spyder
Python ya cargó la versión vieja de la librería en memoria. Para que tome los cambios tienes que reiniciar el kernel. En Spyder: menú Consolas → Reiniciar kernel. En terminal: cierra y abre una nueva.
6
Verificar que el error desapareció
Vuelve a ejecutar el código original:
import wallstreet
stock = wallstreet.Stock(quote="AAPL")
print(stock.price)
238.50
# ✓ sin errores — el precio apareció correctamente
comando adicional — opcional
actualizar wallstreet cuando publiquen el fix oficial Cuando los autores de la librería corrijan esto en una versión nueva, podrás actualizar sin tocar el código manualmente. Ejecuta en tu terminal de Anaconda (con el entorno trading activo):

pip install --upgrade wallstreet

Si la nueva versión ya incluye el fix, no necesitarás el parche manual que hiciste.

Lo que tienes que dominar de esta lección
▸
El error no fue tuyo. Yahoo Finance actualizó su API y wallstreet aún no se adaptó. Es un problema de compatibilidad entre librerías.
▸
Localizar el archivo de una librería: import wallstreet; print(wallstreet.__file__) devuelve la ruta exacta del archivo fuente instalado.
▸
El fix: dentro de wallstreet.py, en la clase Stock, cambiar session=requests.Session() por session=None.
▸
Por qué funciona el fix: con session=None, yfinance gestiona sus propias cookies y sesión — exactamente lo que Yahoo Finance ahora requiere.
▸
Reiniciar el kernel es obligatorio después de editar el archivo. Python tiene la versión vieja en memoria y no tomará los cambios hasta que se limpie.
▸
Fix permanente futuro: pip install --upgrade wallstreet cuando los autores publiquen la versión corregida.
