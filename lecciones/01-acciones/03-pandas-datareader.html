03_pandas_datareader.py — Procesamiento de Datos Financieros
¿Qué es pandas_datareader y para qué sirve?
contexto pandas_datareader es una librería que actúa como puente entre Python y distintas fuentes de datos financieros. En lugar de aprender cómo conectarte a cada API por separado, usas siempre la misma sintaxis y la librería se encarga del resto. No es la opción más rápida, pero es muy conocida — la encontrarás frecuentemente en blogs de inversión y en código financiero de internet.
Las cuatro librerías de esta lección
pandas_datareader

¿por qué usamos Stooq? Es la única fuente sin API key que actualmente funciona. Yahoo Finance también estaría disponible sin key, pero lleva tiempo sin responder correctamente a través de esta librería. Si necesitas datos de Yahoo, usa directamente yfinance como vimos en lecciones anteriores.
¿qué es Stooq? Una plataforma financiera en línea que provee datos bursátiles históricos de manera gratuita. No es tan conocida como Yahoo Finance, pero es confiable y no requiere registro ni token para acceder.
Dos formas de descargar los mismos datos
¿por qué dos métodos? pandas_datareader ofrece una función rápida (get_data_stooq) y también acceso directo a la clase interna (StooqDailyReader). Ambos traen los mismos datos — la diferencia está en cómo los recibes y cuánto control tienes sobre el proceso.
método 1 — función directa (la más simple)
# Definir rango de fechas con datetime
start_date = datetime.datetime(2020, 1, 1)
end_date = datetime.datetime(2025, 1, 1)

# Petición directa a Stooq
df_stooq = pdr.get_data_stooq(
    symbols = ["AMZN"],
    start = start_date,
    end = end_date
)
¿qué hace cada parámetro? symbols — lista de tickers a descargar. Siempre entre corchetes aunque sea uno solo.
start — fecha de inicio. Aquí se pasa un objeto datetime (no un string).
end — fecha de fin. Mismo formato que start.

Diferencia importante con yfinance: aquí las fechas son objetos datetime, no cadenas de texto como "2020-01-01".

método 2 — clase directa (más control)
# Usando la clase interna StooqDailyReader
stooq_df = pdr.stooq.StooqDailyReader(
    symbols = ["AMZN"],
    start = "2020-01-01", # aquí sí acepta strings
    end = "2025-01-01"
)
df = stooq_df.read() # ejecutar la lectura
df = df[::-1] # invertir el orden (de antiguo a reciente)
¿qué es [::-1]? Es slicing de Python aplicado a un DataFrame. Significa: "dame todos los elementos, pero al revés". Es equivalente a voltear una lista de cabeza. Stooq devuelve los datos del más reciente al más antiguo — con [::-1] los invertimos para que queden cronológicamente correctos (de antiguo a reciente), que es el formato que espera la librería de gráficas.
método 1 — get_data_stooq()
Función directa
Una sola línea. Fechas con datetime. Resultado inmediato en un DataFrame. Menos control sobre el proceso.
método 2 — StooqDailyReader()
Clase + .read()
Dos pasos: crear el lector y luego llamar .read(). Acepta strings como fechas. Más flexible para personalización.
doble nivel en columnas — problema que resolveremos Stooq devuelve columnas con dos niveles: el nombre de la columna (Open, High, Low...) y el ticker (AMZN). Ejemplo: (Open, AMZN), (High, AMZN). Esto es útil si descargas varios tickers a la vez, pero para graficar un solo activo es un obstáculo que hay que eliminar.
Limpiar el DataFrame antes de graficar
¿por qué necesitamos limpiar? mplfinance espera un DataFrame con columnas llamadas exactamente Open, High, Low, Close, Volume — en un solo nivel. Si el DataFrame tiene dos niveles de columnas como (Open, AMZN), la librería de gráficas no lo reconocerá y lanzará un error.
El problema: columnas con doble nivel
# así llegan las columnas desde Stooq:
MultiIndex([( 'Open', 'AMZN'),
( 'High', 'AMZN'),
( 'Low', 'AMZN'),
('Close', 'AMZN'),
('Volume','AMZN')],)

# nivel 0: nombre de la columna → queremos conservar esto
# nivel 1: nombre del ticker → esto hay que eliminar
la solución — droplevel()
df.columns = df.columns.droplevel(level=1)
¿qué hace droplevel(level=1)? droplevel() elimina un nivel de las columnas. level=1 indica que queremos eliminar el segundo nivel (el del ticker). Los niveles se cuentan desde 0, así que:

level=0 → nombre de la columna (Open, High...) — conservar
level=1 → nombre del ticker (AMZN) — eliminar
# resultado después de droplevel(level=1):
Index(['Open', 'High', 'Low', 'Close', 'Volume'])

✓ Un solo nivel — listo para graficar
Flujo completo de limpieza
1
Descargar con StooqDailyReader
Los datos llegan en orden inverso y con columnas de doble nivel.
df = pdr.stooq.StooqDailyReader(symbols=["AMZN"], start="2020-01-01", end="2025-01-01").read()
2
Invertir el orden cronológico
De más reciente → más antiguo, pasar a más antiguo → más reciente.
df = df[::-1]
3
Eliminar el nivel extra de columnas
Quitar el nivel del ticker para dejar solo los nombres de columna.
df.columns = df.columns.droplevel(level=1)
¿cuándo NO necesitas droplevel? Si usas get_data_stooq() con un solo ticker, igual obtendrás el doble nivel. Si descargas varios tickers a la vez (ej. ["AMZN", "TSLA", "MSFT"]), el doble nivel es útil — te permite separar los datos por empresa usando df["Open"]["AMZN"]. Solo elimínalo cuando vayas a graficar un activo individual.
Visualizar datos con gráfica de velas (candlestick)
¿qué es una gráfica de velas? Es el estándar visual del trading. Cada "vela" representa un período de tiempo (aquí, un día) y contiene cuatro datos: el precio de apertura, el máximo, el mínimo y el cierre. De un vistazo puedes ver tendencias, volatilidad y movimientos del mercado mucho mejor que con una línea simple.
Anatomía de una vela
Verde (alcista): el precio cerró más alto que como abrió → el mercado subió ese día.
Rojo (bajista): el precio cerró más bajo que como abrió → el mercado bajó ese día.
Mecha: la línea delgada muestra el máximo y mínimo del día, incluyendo movimientos fuera del cuerpo.
el código de la gráfica
mpf.plot(
    data = df,
    style = "yahoo", # estilo visual: fondo blanco, colores financieros
    type = "candle", # gráfica de velas (alternativas: "line", "ohlc")
    volume = True, # mostrar panel de volumen debajo
    title = "Precio de Amazon",
    figsize = (22, 6), # ancho=22, alto=6 pulgadas
    warn_too_much_data = df.shape[0], # silencia la advertencia de >500 velas
    figscale = 3.0, # escala general: velas más gruesas y visibles
    panel_ratios = (7, 3) # 70% precios, 30% volumen
)
plt.show()
¿Qué hace cada parámetro?
ParámetroValor en la lección¿Para qué sirve?
data df El DataFrame limpio con columnas Open, High, Low, Close, Volume.
style "yahoo" Estilo visual predefinido. Hay otros: "default", "charles", "nightclouds".
type "candle" Tipo de gráfica. Alternativas: "line" (línea simple), "ohlc" (barras).
volume True Añade un panel secundario con el volumen diario de operaciones.
figsize (22, 6) Tamaño en pulgadas (ancho, alto). Número grande = imagen más grande.
warn_too_much_data df.shape[0] Silencia el aviso cuando hay más de 500 velas. df.shape[0] = número de filas.
figscale 3.0 Escala todo el gráfico. Valores mayores hacen las velas más gruesas y legibles.
panel_ratios (7, 3) Proporción de altura entre los dos paneles. 7 partes de precio, 3 de volumen.
¿qué es df.shape[0]? df.shape devuelve una tupla con las dimensiones del DataFrame: (filas, columnas). Al poner [0] tomamos solo el número de filas — en este caso, la cantidad de días de datos descargados. Si descargaste 5 años de datos de lunes a viernes, serán aproximadamente 1,260 filas.
ventaja vs. gráfica de línea simple La gráfica de velas muestra cuatro valores por día (Open, High, Low, Close) en lugar de uno solo (Close). Esto permite ver de un vistazo la volatilidad del día, la dirección del mercado y posibles patrones de reversión — información que una línea simple no puede comunicar.

Lo que tienes que dominar de esta lección
▸
pandas_datareader es un puente a múltiples fuentes de datos. Se importa como pdr y la fuente recomendada sin API key es Stooq.
▸
Dos formas de pedir datos a Stooq: pdr.get_data_stooq() (rápida, fechas datetime) y pdr.stooq.StooqDailyReader().read() (más control, acepta strings).
▸
Stooq devuelve datos en orden inverso. Usar df[::-1] para invertirlos a orden cronológico antes de graficar.
▸
Columnas de doble nivel: Stooq incluye el ticker como segundo nivel. Eliminar con df.columns.droplevel(level=1) antes de pasar a mplfinance.
▸
Listar fuentes disponibles: [i for i in dir(pdr) if "get_data" in i] muestra las 10 opciones. Solo Stooq funciona sin API key hoy.
▸
mplfinance genera gráficas de velas financieras. Requiere columnas limpias (un solo nivel) con nombres exactos: Open, High, Low, Close, Volume.
▸
Parámetros clave de mpf.plot(): type="candle", volume=True, panel_ratios=(7,3), warn_too_much_data=df.shape[0].
▸
Limitación de la librería: pandas_datareader es más lenta que yfinance. Úsala cuando encuentres código existente que la use, o cuando necesites una fuente alternativa a Yahoo Finance.
