Fundamentos · 00

¿Qué es yfinance y para qué sirve?

El punto de partida antes de escribir cualquier línea de código

yfinance es una librería de Python que actúa como un puente entre tu código y los datos financieros de Yahoo Finance. Con unas pocas líneas puedes descargar años de precios históricos de cualquier acción, índice o criptomoneda del mundo, de forma completamente gratuita.

🏦 ¿Qué es Yahoo Finance?

Es uno de los proveedores de datos financieros más grandes del mundo. Tiene información histórica y en tiempo casi real de acciones, índices, divisas y más. Normalmente accedes a él por la web — yfinance te permite hacerlo desde Python.

🐍 ¿Qué hace yfinance exactamente?

Envía peticiones a los servidores de Yahoo Finance y convierte la respuesta en un DataFrame de pandas — una tabla ordenada lista para analizar, graficar o procesar en tus estrategias de trading.

Como tener un asistente que va a la biblioteca de datos financieros, busca lo que necesitas, y te lo entrega ya ordenado en una hoja de cálculo.

¿Por qué usamos esta librería en trading?

Para construir cualquier estrategia de trading necesitas datos históricos de precios. Estos datos son la materia prima de todo. Con ellos puedes:

📊 Backtesting

Probar si tu estrategia habría funcionado en el pasado antes de arriesgar dinero real

📈 Análisis técnico

Calcular indicadores como medias móviles, RSI, MACD sobre datos reales

🤖 Machine Learning

Entrenar modelos para predecir comportamientos de precio

📉 Análisis de riesgo

Medir volatilidad, drawdown y otros métricas de riesgo históricas

✦ Instalar yfinance: Si aún no la tienes, abre tu terminal y ejecuta: pip install yfinance
Fundamentos · 01

Librerías y parámetros base

El bloque inicial que va al principio de todo script de esta lección

python — imports y parámetros
# ── Librerías ──────────────────────────────────────────
import yfinance as yf
import pandas as pd
import time
import matplotlib.pyplot as plt

# ── Parámetros base ────────────────────────────────────
ticker       = "AMZN"
fecha_inicio = "2023-01-01"
fecha_final  = "2025-01-01"
intervalo    = "1d"

Línea por línea — Librerías

import yfinance as yf
Importa la librería de Yahoo Finance. El alias yf es solo un apodo para no escribir "yfinance" completo cada vez. Ahora puedes llamarla con yf.download() en lugar de yfinance.download()
import pandas as pd
Pandas es la librería para manipular tablas de datos (DataFrames). Cuando yfinance descarga datos, los devuelve en formato pandas. El alias pd es la convención universal.
import time
Módulo de Python para medir tiempo. En esta lección lo usamos para comparar cuánto tarda una descarga concurrente vs una secuencial. La función time.time() devuelve el tiempo actual en segundos.
import matplotlib.pyplot as plt
Librería para crear gráficas. El alias plt es la convención estándar. Con ella graficaremos los precios de cierre al final de la lección.

Línea por línea — Parámetros

Los parámetros son variables de configuración. Se definen una vez al inicio y se reutilizan en todos los ejemplos. Si quieres cambiar la acción o el rango de fechas, solo lo cambias aquí y todo el script se actualiza automáticamente.

ticker = "AMZN"
El símbolo bursátil de la acción a descargar. AMZN = Amazon. Puedes cambiarlo por cualquier otro: "TSLA", "AAPL", "NVDA", etc.
fecha_inicio = "2023-01-01"
Fecha de inicio del rango de datos. Formato obligatorio: YYYY-MM-DD (año-mes-día). Esta fecha SÍ se incluye en los datos.
fecha_final = "2025-01-01"
Fecha final del rango. Importante: esta fecha NO se incluye en los datos. Los datos llegan hasta el 31 de diciembre de 2024.
intervalo = "1d"
El tamaño de cada vela. "1d" = una fila por día de bolsa. Cada fila resume todo lo que pasó con el precio en ese día (apertura, máximo, mínimo, cierre).
⚠ El error original del curso era escribir "id" en lugar de "1d". Yahoo Finance no reconoce "id" como intervalo válido, por eso fallaba la descarga.
Ejemplo · 02

Datos históricos con rango de fechas

La forma más fundamental de descargar datos: tú dices exactamente qué periodo quieres

código completo · ejemplo 1copiar y pegar
# Ejemplo 1: Descargar datos históricos con rangos de fechas e intervalo diario
df = yf.download(tickers=ticker, start=fecha_inicio, end=fecha_final, interval=intervalo)
print(df)

Ahora, la misma línea de descarga, desglosada parámetro por parámetro:

python — ejemplo 1
df = yf.download(
    tickers  = ticker,        # Qué acción descargar
    start    = fecha_inicio,  # Desde cuándo
    end      = fecha_final,   # Hasta cuándo (no incluida)
    interval = intervalo      # Tamaño de cada vela
)
print(df)

¿Qué hace este ejemplo?

Le pide a Yahoo Finance todos los precios diarios de Amazon desde el 1 de enero de 2023 hasta el 31 de diciembre de 2024. El resultado es un DataFrame donde cada fila es un día de bolsa y cada columna es un tipo de precio.

Línea por línea

df = yf.download(...)
yf.download() es la función principal de yfinance. Hace la petición a Yahoo Finance y devuelve un DataFrame. Lo guardamos en la variable df (abreviación de DataFrame).
tickers = ticker
Le dices qué acción(es) quieres. Aquí pasamos nuestra variable ticker que vale "AMZN". También acepta una lista como ["AMZN", "AAPL"].
start = fecha_inicio
Fecha de inicio del rango. Los datos arrancan desde este día.
end = fecha_final
Fecha final del rango. Importante: esta fecha no aparece en los datos — es como decir "dame todo hasta el día anterior".
interval = intervalo
Define el tamaño de cada fila. Con "1d" cada fila = un día. Con "1h" cada fila = una hora. Con "1m" = un minuto.
print(df)
Imprime la tabla en la consola. Para verla bien formateada en Spyder, haz doble clic en la variable df en el Variable Explorer.

Las columnas del resultado — OHLCV

O
Open
Precio al que abrió la acción ese día
H
High
El precio más alto que alcanzó
L
Low
El precio más bajo que alcanzó
C
Close
Precio al que cerró al final del día
✦ Adj Close recomendado

Precio de cierre ajustado por dividendos y splits. Para análisis histórico y backtesting siempre usa esta columna en lugar de Close simple. El curso tenía versión antigua donde no aparecía — las versiones nuevas sí la incluyen.

📊 Volume

Cuántas acciones se compraron/vendieron ese día. Un volumen alto indica interés elevado. Traders lo usan para confirmar si un movimiento de precio es genuino o artificial.

ℹ Con start/end siempre obtienes los mismos datos sin importar cuándo ejecutas el código. Esto es clave para backtesting reproducible — que tus pruebas siempre usen exactamente los mismos datos.
Ejemplo · 03

Datos intradía con period

Descarga el último mes de datos con velas de 15 minutos — ideal para estrategias durante la sesión del día

código completo · ejemplo 2copiar y pegar
# Ejemplo 2: Descargar datos para el último mes con intervalo de 15 minutos
df = yf.download(tickers=ticker, period="1mo", interval="15m")
print(df)

Ahora, la misma línea de descarga, desglosada parámetro por parámetro:

python — ejemplo 2
df = yf.download(
    tickers  = ticker,    # La misma acción (AMZN)
    period   = "1mo",   # Último mes desde hoy
    interval = "15m"    # Una vela cada 15 minutos
)
print(df)

¿Qué hace este ejemplo?

Descarga los datos del último mes de actividad bursátil, pero con mucho más detalle que el ejemplo 1. En lugar de una fila por día, ahora tienes una fila cada 15 minutos — eso es mucho más granular y permite analizar el comportamiento del precio durante cada sesión.

Línea por línea

period = "1mo"
Remplaza a start/end. En lugar de dar un rango fijo de fechas, le dices "dame el último mes desde hoy". Es una ventana relativa que cambia cada vez que ejecutas el código. Valores válidos: "1d", "5d", "1mo", "3mo", "6mo", "1y", "2y", "5y", "max"
interval = "15m"
Cada fila del DataFrame representa 15 minutos de precio. Si el mercado abre a las 9:30am y cierra a las 4pm, eso son 26 velas por día de bolsa.

period vs start/end — ¿cuándo usar cada uno?

start / end fijo

Siempre devuelve los mismos datos. El código corrido hoy o en 2027 da idéntico resultado. Úsalo para backtesting y cuando necesitas reproducibilidad.

period relativo

Siempre trae los datos más recientes desde hoy. Úsalo para monitoreo en tiempo real, dashboards, o cuando quieres datos actuales.

¿Qué son datos intradía?

"Intraday" significa "dentro del mismo día". Son datos con intervalos de minutos u horas, en lugar de días. Se usan para estrategias de trading de corto plazo donde operas durante la sesión, no entre días.

El ejemplo 1 es como ver el resumen de temperatura diaria de un mes. El ejemplo 2 es como ver la temperatura cada 15 minutos — mucho más detalle, pero solo del mes más reciente.
⚠ Los datos intradía tienen límites de historia. Con "15m" solo puedes pedir hasta 60 días hacia atrás. Si intentas más, Yahoo Finance no te los da. Ver tabla de limitaciones en la sección 08.
Ejemplo · 04

Pre y Post mercado

El ejemplo más complejo de la lección — descarga datos fuera del horario regular y los segmenta por tipo de sesión

ℹ Este ejemplo tiene 4 partes. Léelas en orden porque cada una depende de la anterior.
código completo · ejemplo 3copiar y pegar
# Ejemplo 3: Descargar datos de los últimos 2 días con pre y post mercado en intervalo de 1 minuto
df = yf.download(tickers=ticker, period="2d", interval="1m", prepost=True, progress=False)
print("Cantidad de datos descargados:", df.shape[0])

# Separar Datos del Horario Regular y del Horario Extendido:

# Convertir el Índice del DataFrame a la Zona Horaria de Nueva York
print("Timezone Actual:", df.index.tz)
if df.index.tz is None:
    df.index = pd.to_datetime(df.index).tz_localize("UTC").tz_convert("America/New_York")
else:
    df.index = df.index.tz_convert("America/New_York")

# Definir el rango del horario normal, pre-market y post-market
horario_pre_mercado_inicio = "04:00:00"
horario_pre_mercado_final = "09:30:00"
horario_normal_inicio = "09:30:00"
horario_normal_final = "16:00:00"
horario_post_mercado_inicio = "16:00:00"
horario_post_mercado_final = "20:00:00"

# Separar los datos según el horario
datos_pre_mercado = df.between_time(start_time=horario_pre_mercado_inicio, end_time=horario_pre_mercado_final).iloc[:-1]
datos_horario_normal = df.between_time(start_time=horario_normal_inicio, end_time=horario_normal_final)
datos_post_mercado = df.between_time(start_time=horario_post_mercado_inicio, end_time=horario_post_mercado_final).iloc[1:]

# Mostrar los datos
print("Datos del horario normal:")
print(datos_horario_normal)

print("\nDatos del pre mercado:")
print(datos_pre_mercado)

print("\nDatos del post mercado:")
print(datos_post_mercado)

Ahora, el mismo código dividido en sus 4 partes, explicadas una por una:

Parte A — Descargar los datos

python — descarga con prepost
df = yf.download(
    tickers  = ticker,
    period   = "2d",      # Solo los últimos 2 días
    interval = "1m",      # Una vela por minuto
    prepost  = True,     # Incluir pre y post mercado
    progress = False     # No mostrar barra de progreso
)
print("Cantidad de datos descargados:", df.shape[0])
period = "2d"
Solo los últimos 2 días. Se usa period en lugar de start/end para siempre traer los más recientes.
interval = "1m"
Una vela por minuto — el nivel más granular disponible. Solo disponible para los últimos 8 días.
prepost = True
El parámetro clave de este ejemplo. Por defecto yfinance solo descarga el horario regular (9:30am–4pm). Con True también incluye el pre-market (4am–9:30am) y post-market (4pm–8pm).
progress = False
Desactiva la barra de progreso que normalmente imprime en consola. Útil cuando tienes muchas descargas y no quieres llenar la consola de texto.
df.shape[0]
df.shape devuelve (filas, columnas). El índice [0] toma solo el número de filas — es decir, cuántas velas de 1 minuto descargaste en total entre los 2 días.

Parte B — Convertir zona horaria

python — conversión de timezone
print("Timezone actual:", df.index.tz)   # Muestra "UTC"

if df.index.tz is None:
    # Si no tiene zona horaria asignada: primero localizar, luego convertir
    df.index = pd.to_datetime(df.index).tz_localize("UTC").tz_convert("America/New_York")
else:
    # Si ya tiene zona horaria: solo convertir
    df.index = df.index.tz_convert("America/New_York")
df.index.tz
Muestra la zona horaria del índice de fechas del DataFrame. Normalmente dice "UTC" — la hora universal estándar.
if df.index.tz is None
Verificamos si el índice ya tiene zona horaria asignada. Si es None, no tiene. Esto puede variar según la versión de yfinance.
tz_localize("UTC")
Cuando el índice no tiene zona horaria, primero le asignamos UTC (decimos "estos números representan hora UTC").
tz_convert("America/New_York")
Convierte de UTC a hora de Nueva York. Necesario porque el mercado americano opera en ese huso horario y necesitamos que los filtros de hora funcionen correctamente.
UTC es como el reloj maestro del mundo. Nueva York tiene 5 horas de diferencia (en invierno). Si no conviertes, un evento a las 9:30am NY aparecería como 14:30 en UTC y tus filtros no funcionarían.

Parte C — Definir los horarios

python — definición de rangos horarios
# Pre-market: antes de que abra la bolsa
horario_pre_mercado_inicio = "04:00:00"
horario_pre_mercado_final  = "09:30:00"

# Horario regular: la bolsa está abierta
horario_normal_inicio = "09:30:00"
horario_normal_final  = "16:00:00"

# Post-market: después de cerrar
horario_post_mercado_inicio = "16:00:00"
horario_post_mercado_final  = "20:00:00"

Son simples strings con horas en formato HH:MM:SS. Representan los tres bloques del día bursátil americano:

4:00am – 9:29am
Pre-market Antes de abrir. Menor volumen, spreads más amplios, mayor riesgo. Solo grandes instituciones y traders avanzados operan aquí. Muy sensible a noticias nocturnas.
9:30am – 4:00pm
Horario regular La bolsa está oficialmente abierta. Máxima liquidez y participación. Es donde se produce la mayor parte del volumen del día.
4:01pm – 8:00pm
Post-market / After Hours Después del cierre. Similar al pre-market. Importante cuando hay resultados trimestrales o noticias después del cierre.

Parte D — Separar los datos

python — filtrado por horario
# Pre-market: de 4am a 9:30am
# .iloc[:-1] quita el último registro (9:30am ya es horario normal)
datos_pre_mercado = df.between_time(
    horario_pre_mercado_inicio, horario_pre_mercado_final
).iloc[:-1]

# Horario regular: de 9:30am a 4pm
datos_horario_normal = df.between_time(
    horario_normal_inicio, horario_normal_final
)

# Post-market: de 4pm a 8pm
# .iloc[1:] quita el primer registro (4:00pm ya está en horario normal)
datos_post_mercado = df.between_time(
    horario_post_mercado_inicio, horario_post_mercado_final
).iloc[1:]
df.between_time(inicio, fin)
Filtra el DataFrame por rango de hora, ignorando la fecha. Si tu DataFrame tiene datos de varios días, between_time("09:30", "16:00") trae todas las filas de todos los días donde la hora esté en ese rango.
.iloc[:-1]
Selecciona "todos los registros excepto el último". En el pre-market, el rango va hasta "09:30:00" — pero ese minuto exacto ya pertenece al horario normal. iloc[:-1] lo elimina del pre-market para evitar duplicados.
.iloc[1:]
Selecciona "todos los registros excepto el primero". El post-market empieza a las "16:00:00" — ese minuto ya pertenece al cierre del horario normal. iloc[1:] lo elimina del post-market.
✦ ¿Para qué sirve separar los horarios? Muchos traders solo operan en horario regular para evitar la baja liquidez. Otros buscan oportunidades en pre-market cuando hay noticias importantes. Tener los datos separados permite analizar cada sesión por su cuenta.
Ejemplos · 05

Múltiples activos

Descargar varias acciones a la vez — y medir si vale la pena hacerlo en paralelo

código completo · ejemplo 4 (concurrente)copiar y pegar
# Ejemplo 4: Descargar Datos para múltiples activos (Descarga Concurrente)
tickers = ["AMZN", "AAPL", "MSFT", "TSLA", "NFLX", "META", "XYZ", "NVDA", "PYPL", "QCOM"]

tiempo_inicio = time.time()
df_activos = yf.download(tickers=tickers, start=fecha_inicio, end=fecha_final, interval=intervalo, threads=True)
tiempo_final = time.time()
print("La descarga tomó {} segundos".format(tiempo_final - tiempo_inicio))
print(df_activos)
código completo · ejemplo 5 (secuencial)copiar y pegar
# Ejemplo 5: Descargar Datos para múltiples activos (Descarga Secuencial)
tiempo_inicio = time.time()
df_activos = yf.download(tickers=tickers, start=fecha_inicio, end=fecha_final, interval=intervalo, threads=False)
tiempo_final = time.time()
print("La descarga tomó {} segundos".format(tiempo_final - tiempo_inicio))
print(df_activos)

Ahora, ambos ejemplos juntos para comparar tiempos, desglosados:

python — ejemplos 4 y 5
# Lista de 10 acciones a descargar
tickers = ["AMZN", "AAPL", "MSFT", "TSLA", "NFLX",
           "META", "XYZ",  "NVDA", "PYPL", "QCOM"]

# ── Ejemplo 4: Descarga CONCURRENTE (paralela) ─────────
tiempo_inicio = time.time()   # Marca el tiempo antes
df_activos = yf.download(
    tickers  = tickers,
    start    = fecha_inicio,
    end      = fecha_final,
    interval = intervalo,
    threads  = True   # Descarga varias al mismo tiempo
)
tiempo_final = time.time()    # Marca el tiempo después
print("La descarga tomó {} segundos".format(tiempo_final - tiempo_inicio))

# ── Ejemplo 5: Descarga SECUENCIAL (una por una) ───────
tiempo_inicio = time.time()
df_activos = yf.download(
    tickers  = tickers,
    start    = fecha_inicio,
    end      = fecha_final,
    interval = intervalo,
    threads  = False  # Descarga una por una
)
tiempo_final = time.time()
print("La descarga tomó {} segundos".format(tiempo_final - tiempo_inicio))

Línea por línea

tickers = [...]
En lugar de un solo string, pasamos una lista de símbolos bursátiles. yfinance acepta listas y descarga todas de una sola llamada.
tiempo_inicio = time.time()
time.time() devuelve el tiempo actual en segundos desde una referencia fija. Lo guardamos antes de descargar para medir cuánto tarda.
threads = True
Descarga concurrente. yfinance lanza múltiples peticiones al mismo tiempo, una por cada acción. Para listas grandes (100+), esto puede ser significativamente más rápido.
threads = False
Descarga secuencial. Descarga AMZN, espera, descarga AAPL, espera, etc. Para 10 acciones la diferencia con True es mínima.
tiempo_final - tiempo_inicio
Resta el tiempo final menos el inicial. El resultado son los segundos que tardó la descarga.
"... {}".format(valor)
Forma de insertar un valor dentro de un string. Los {} son marcadores que se reemplazan con el valor que pasas en .format().

¿Cuándo usar threads=True?

⚡ Concurrente — threads=True

Descarga todas al mismo tiempo. Igual que abrir 10 pestañas del navegador a la vez. Para listas grandes (50-200 activos) puede ser 5-10x más rápido.

➡ Secuencial — threads=False

Descarga una por una. Más simple, menos carga al servidor. Para 10 activos la diferencia con True es casi imperceptible.

Secuencial = un cajero atendiendo 10 clientes en fila. Concurrente = 10 cajeros atendiendo 10 clientes al mismo tiempo. Con pocos clientes la diferencia es mínima. Con 200 clientes, enorme.

El DataFrame con múltiples activos

Cuando descargas varias acciones, las columnas tienen dos niveles: el tipo de precio arriba y el ticker abajo. Por ejemplo, para acceder al Close de Apple: df_activos["Close"]["AAPL"]

Ejemplo · 06

Máximo histórico sin multi-nivel

Descargar todo el historial disponible y simplificar la estructura de columnas

código completo · ejemplo 6copiar y pegar
# Ejemplo 6: Descargar el máximo histórico de datos sin multi nivel
df = yf.download(tickers=ticker, period="max", interval=intervalo, multi_level_index=False)
print(df)

Ahora, la misma línea de descarga, desglosada parámetro por parámetro:

python — ejemplo 6
df = yf.download(
    tickers           = ticker,
    period            = "max",   # Todo el historial de Amazon
    interval          = intervalo,
    multi_level_index = False   # Columnas simples sin doble nivel
)
print(df)

Línea por línea

period = "max"
Descarga todo el historial disponible en Yahoo Finance para esa acción. Amazon lleva décadas en bolsa, así que obtienes registros desde los años 90 hasta hoy. Solo funciona con intervalos diarios o superiores (sin límite de tiempo).
multi_level_index = False
Elimina el doble nivel de columnas. Por defecto, incluso con un solo ticker, yfinance genera columnas con dos niveles (tipo de precio + ticker). Con False, las columnas quedan simples: solo Open, High, Low, Close, Volume.

Con vs sin multi_level_index

Sin multi_level_index=False doble nivel

Para acceder al Close: df["Close"]["AMZN"]
Más verboso. Puede complicar el código de tu sistema de trading.

Con multi_level_index=False simple

Para acceder al Close: df["Close"]
Más limpio. El instructor lo recomienda para sistemas de trading propios.

✦ Consejo del instructor: Usa siempre multi_level_index=False cuando trabajes con un solo activo. El código queda más limpio y el acceso a columnas es más rápido de escribir.

Limitaciones de Yahoo Finance

Lo que sí y no puedes pedir según el intervalo — esencial para diseñar sistemas de trading

Yahoo Finance es gratuita pero tiene restricciones. El intervalo que eliges define cuánto historial puedes pedir. Ignorar esto provoca descargas vacías o errores.

Glosario completo

Todos los términos de la lección explicados desde cero

El mercado

Acción Stock / Share
Una pequeña parte de la propiedad de una empresa. Si compras una acción de Amazon eres dueño de una fracción microscópica de la compañía. Si la empresa vale más, tu fracción también.
Como si una empresa fuera una pizza cortada en millones de rebanadas. Puedes comprar una o más rebanadas.
Ticker / Símbolo bursátil Ticker symbol
El apodo corto y único de cada empresa en la bolsa. AMZN = Amazon, AAPL = Apple, TSLA = Tesla, NVDA = Nvidia. Se usa en el código para identificar qué datos descargar.
Bolsa de valores Stock Exchange
El lugar (o plataforma digital) donde se compran y venden acciones. La más grande del mundo es la NYSE en Nueva York. Opera de lunes a viernes en horario laboral.
Liquidez Liquidity
Qué tan fácil es comprar o vender algo sin mover mucho su precio. Muchos compradores y vendedores = alta liquidez = más fácil y barato operar. El pre/post mercado tiene poca liquidez.
Como vender un iPhone (fácil, muchos compradores) vs. vender una pieza de arte rarísima (difícil, pocos interesados).
Spread Bid-Ask Spread
La diferencia entre el precio al que alguien quiere vender y el que otro quiere pagar. Un spread amplio = más caro operar. En pre/post mercado el spread es mayor porque hay poca liquidez.
Como en una casa de cambio: te compran dólares a 17 pero te los venden a 17.50. Esa diferencia de 50 centavos es el spread.

Los datos

Vela / Candlestick
Un resumen del precio de una acción durante un periodo de tiempo. Si el intervalo es "1d", cada vela resume todo lo que pasó con el precio ese día: a qué precio abrió, hasta dónde subió, hasta dónde bajó y a qué precio cerró.
Open (O)
El precio al que abrió la acción ese día/periodo. Es el primer precio al que se realizó una transacción cuando abrió el mercado.
High (H)
El precio más alto que alcanzó la acción durante ese periodo. El "techo" de ese día.
Low (L)
El precio más bajo que tocó la acción durante ese periodo. El "piso" de ese día.
Close (C)
El precio al que cerró la acción al final del periodo. El último precio al que se realizó una transacción antes de cerrar el mercado.
Adj Close recomendado
El precio de cierre ajustado hacia atrás en el tiempo para que sea comparable. Corrige el precio histórico cuando la empresa paga dividendos o hace un split (divide sus acciones), eliminando saltos artificiales en la gráfica. Úsalo siempre para backtesting.
Volume
Cuántas acciones se compraron y vendieron durante ese periodo. Volumen alto = mucho interés. Los traders lo usan para confirmar si un movimiento de precio es genuino: precio que sube con volumen alto es más creíble que uno que sube con volumen bajo.

Tiempo e intervalos

Intervalo Interval / Timeframe
Cuánto tiempo representa cada fila del DataFrame. Con "1d" cada fila = un día. Con "15m" cada fila = 15 minutos. Menor intervalo = más datos por día, pero con más restricciones de historia disponible.
Datos intradía Intraday
Datos con intervalos en minutos u horas (dentro del mismo día). Se usan para estrategias de trading de corto plazo donde operas durante la sesión del día, no entre días.
Backtesting
Probar una estrategia de trading usando datos históricos para ver si hubiera funcionado en el pasado. Antes de arriesgar dinero real, simulas tu estrategia en datos que ya ocurrieron para medir su rentabilidad. Requiere datos fijos (start/end), no relativos (period).
Como revisar los resultados de los últimos 5 años antes de apostar en el casino, para ver si tu estrategia habría ganado históricamente.

Conceptos de código

DataFrame
Una tabla de datos en Python (librería pandas). Filas y columnas, como una hoja de Excel pero en código. Cuando descargas datos con yf.download(), recibes un DataFrame. Cada fila es un periodo de tiempo, cada columna es un dato (Open, Close, Volume, etc.)
df.index
La columna especial que identifica cada fila del DataFrame. En datos financieros, el índice siempre son las fechas/horas. Es sobre lo que funcionan métodos como between_time().
UTC
Coordinated Universal Time — la hora estándar global usada como referencia. Yahoo Finance entrega los datos en UTC. Como el mercado americano opera en hora de Nueva York, hay que convertir de UTC a "America/New_York" para que los filtros de horario funcionen correctamente.
multi_level_index
Controla si las columnas del DataFrame tienen uno o dos niveles de encabezado. Con False, las columnas son simples (Open, Close, Volume). Recomendado para un solo activo — hace el código más limpio.
Descarga concurrente vs secuencial
Concurrente (threads=True): descarga varias acciones al mismo tiempo en paralelo. Secuencial (threads=False): descarga una por una. Para listas pequeñas la diferencia es mínima. Para 100+ activos la descarga concurrente puede ser 5-10x más rápida.
